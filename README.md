# ICS344 Information Security Project

## Project Overview
The goal of this project is to analyze, exploit, and fix security vulnerabilities in a real-world serverless cloud application using the OWASP DVSA (Damn Vulnerable Serverless Application).

The application is deployed on AWS and follows a serverless architecture that includes Amazon API Gateway, AWS Lambda functions, and backend services such as DynamoDB and S3. 

---

## Objectives
The main objectives of this project are:
- Identify and exploit security vulnerabilities in DVSA
- Understand how these vulnerabilities affect serverless applications
- Apply proper fixes using secure design principles
- Verify that vulnerabilities are resolved without breaking functionality


---

## Technologies Used
- AWS Lambda
- Amazon API Gateway
- Amazon S3
- Amazon DynamoDB
- Amazon Cognito (JWT Authentication)
- CloudWatch Logs

## Covered Vulnerabilities
This project covers the 10 DVSA vulnerability lessons:

1. Event Injection  
2. Broken Authentication  
3. Sensitive Data Exposure  
4. Insecure Cloud Configuration  
5. Broken Access Control  
6. Denial of Service (DoS)  
7. Over-Privileged Functions  
8. Logic Vulnerabilities  
9. Vulnerable Dependencies  
10. Unhandled Exceptions
