# Logic Vulnerabilities (Race Condition)

The Logic Vulnerability in the DVSA application occurs due to a race condition in the order processing workflow. The billing function does not lock the order while processing payment, which allows an attacker to modify the order at the same time.

This means an attacker can pay for one item but receive more items, because the system updates the order quantity after billing has already started.

---

## Vulnerability Exploiting

To achieve this vulnerability, you have to follow these steps:

1. Log in to the DVSA application  
2. Add one item to the cart (quantity = 1)  
3. Proceed to checkout page  

---

### Capture Required Data

1. Open Browser DevTools → Network tab  
2. Select the order request  
3. Copy:
   - `order-id` from the Preview tab  
   - `Authorization token` from the Headers tab  

---

### Run the Exploit

1. Create a Python script using threading  
2. The script should send two requests at the same time:
   - Request 1: billing request (pay for 1 item)  
   - Request 2: update request (change quantity to 5) with a small delay (0.1s)  
3. Run the script using PowerShell  

---

## Proof of Exploit

After running the attack:

- The billing request succeeds for one item ($33)  
- The update request changes the quantity to 5  

When checking the order:

- Quantity becomes 5  
- Total amount remains $33  

This confirms that the system charged for one item but delivered five items.

---

## Vulnerability Fixing

The fix was applied by locking the order status at the start of the billing process.

To apply the fix:

1. Open the billing Lambda function (DVSA-ORDER-BILLING)  
2. Modify the billing logic  
3. Set the order status to a temporary value (e.g., 115 = billing in progress) at the beginning  
4. Reject any update requests while billing is in progress  

---

## Verification After Fix

To verify that the fix is working:

1. Run the same exploit again  
2. Observe the results  

After the fix:

- The update request is rejected  
- The system returns a message like:

order already paid

- The quantity remains 1  
- The total amount remains correct ($33)  

This confirms that the race condition is fixed and the order cannot be modified during billing.
